# CSS文件结构说明

## 概述
本项目的CSS已经重构为模块化结构，将原本的单个`style.css`文件拆分为多个功能模块，提高了代码的可维护性和可读性。

## 文件结构

### 核心模块文件
- **`main.css`** - 主入口文件，导入所有其他CSS模块
- **`base.css`** - 基础样式（CSS变量、重置样式、全局样式、动画）
- **`layout.css`** - 布局样式（容器、网格、间距、响应式布局工具类）
- **`header.css`** - 顶部导航栏样式
- **`components.css`** - 通用组件样式（按钮、表单、卡片、提示消息等）
- **`footer.css`** - 页脚样式

### 页面专用模块
- **`pages/index.css`** - 首页专用样式
- **`pages/auth.css`** - 认证页面样式（登录、注册）
- **`pages/goods.css`** - 商品相关页面样式
- **`pages/cart.css`** - 购物车页面样式

### 旧文件
- **`style.css`** - 原始CSS文件（已拆分，保留作为备份）

## 使用方法

在JSP页面中引用CSS：
```html
<link rel="stylesheet" href="css/main.css">
```

## CSS变量

主要颜色变量定义在 `base.css` 中：
- `--primary-color`: #5D4037 (主色 - 深棕色)
- `--secondary-color`: #FF9800 (辅助色 - 橙色)
- `--accent-color`: #FF5722 (强调色 - 深橙色)
- `--success-color`: #4CAF50 (成功色 - 绿色)
- `--error-color`: #F44336 (错误色 - 红色)
- `--warning-color`: #FF9800 (警告色 - 橙色)

## 优势

1. **模块化**: 每个文件专注于特定功能
2. **可维护性**: 易于查找和修改特定样式
3. **可重用性**: 组件样式可以在多个页面中使用
4. **一致性**: 统一的颜色和样式变量
5. **性能优化**: 可按需加载特定模块
6. **团队协作**: 不同开发者可以并行开发不同模块

## 响应式设计

所有CSS模块都包含响应式设计，支持以下断点：
- 768px: 平板设备
- 480px: 手机设备
- 992px: 小型桌面设备

## 浏览器兼容性

支持现代浏览器：
- Chrome 60+
- Firefox 55+
- Safari 12+
- Edge 79+
